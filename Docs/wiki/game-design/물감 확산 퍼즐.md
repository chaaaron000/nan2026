# 물감 확산 퍼즐

> Sources: 물감 확산 퍼즐 기획서, Unknown
> Raw: [물감 확산 퍼즐 기획서](../../raw/Notion/물감%20확산%20퍼즐%20기획서%203a2a3b111a0c805d849be9d5c54ffa84.md)

## Overview

물감 확산 퍼즐은 정답 그림과 벽 구조를 관찰한 뒤, 범위와 색상이 정해진 물감통을 격자에 순서대로 배치해 목표 무늬를 재현하는 Unity 2D 기반 PC WebGL 논리 퍼즐이다. 핵심은 완성 상태에서 물감의 시작 위치·도달 범위·사용 순서를 역으로 추론하는 데 있다. 실패나 시간 제한은 없으며, 배치 과정이 아니라 최종 격자 상태만 정답과 같으면 성공한다.

## 핵심 플레이 구조

화면은 같은 크기와 벽 구조를 공유하는 두 격자로 구성된다.

- **정답판:** 목표 색상 무늬와 벽을 표시한다.
- **플레이판:** 플레이어가 물감통을 배치하고 결과를 확인한다.
- **물감통 목록:** 스테이지에서 제공하는 색상, 범위, 남은 수량을 표시한다.
- **보조 기능:** 직전 행동을 복원하는 되돌리기와 시작 상태로 돌아가는 초기화를 제공한다.

한 번의 행동은 `물감통 선택 → 칸 선택 → 물감통 소모 → 확산 계산 및 연출 → 색상 반영 → 정답 비교` 순서로 진행된다. 확산 연출 중에는 다음 입력을 막는다. 모든 칸의 최종 표시 색상이 정답판과 일치하면 남은 물감통 수, 배치 위치, 배치 순서와 관계없이 스테이지를 완료한다.

## 확산 규칙

범위가 `N`인 물감통은 배치 칸을 거리 0으로 보며, 벽을 통과하지 않고 상하좌우로 `N-1`번 이하 이동해 도달 가능한 모든 칸에 적용된다.

- 대각선 이동은 허용하지 않는다.
- 격자 외곽과 셀 사이의 벽은 이동 불가 경계다.
- 정면이 막혀도 남은 이동 횟수 안에 열린 길로 우회할 수 있다.
- 같은 칸에 여러 물감통을 차례로 배치할 수 있다.
- 사용한 물감통은 즉시 소모되고 격자 위 오브젝트로 남지 않는다.

기획서에서는 이를 “맨해튼 거리 확산”이라고 부르지만, 벽을 우회하는 경로까지 허용하므로 구현 기준은 단순 좌표 차가 아니라 **벽을 반영한 격자 그래프의 최단 경로 거리**다. 배치 셀에서 너비 우선 탐색(BFS)을 수행하고 깊이가 `N-1` 이하인 셀을 수집하면 규칙과 일치한다.

## 색상 상태와 혼합

각 셀은 최종 색 이름 대신 R, G, B의 포함 여부를 독립적으로 저장한다. 같은 원색을 여러 번 적용해도 상태는 변하지 않는다.

| R | G | B | 표시 결과 |
|---|---|---|---|
| 0 | 0 | 0 | 빈 칸 |
| 1 | 0 | 0 | 빨강 |
| 0 | 1 | 0 | 초록 |
| 0 | 0 | 1 | 파랑 |
| 1 | 1 | 0 | 노랑 |
| 0 | 1 | 1 | 청록 |
| 1 | 0 | 1 | 자홍 |
| 1 | 1 | 1 | 흰색 |

W는 혼합색이 아니라 지우개 역할을 하는 특수 물감이다. W가 도달한 셀은 R, G, B 상태를 모두 제거해 빈 칸이 되며, 이후 다른 색을 다시 적용할 수 있다. 따라서 `넓게 칠하기 → 일부 지우기 → 필요한 색 덧칠하기` 같은 순서 의존 풀이가 가능하다. 시각적으로는 흰색(`R+G+B`)과 빈 칸(`0+0+0`)이 명확히 구분되어야 한다.

## 퍼즐이 만드는 사고 과정

1. 정답 무늬와 벽 구조를 관찰한다.
2. 각 색 영역을 만들 수 있는 물감통의 후보 위치와 범위를 역산한다.
3. 벽에 막히거나 우회해 도달하는 셀을 계산한다.
4. 색 혼합과 W의 삭제 효과를 고려해 행동 순서를 정한다.
5. 배치 결과를 즉시 확인하고 되돌리기나 초기화로 가설을 수정한다.

위치와 순서를 동시에 다루면서도, 최종 결과만 일치하면 여러 풀이를 허용하는 것이 핵심 차별점이다. 개발자가 기준 풀이를 준비하더라도 더 적은 물감통이나 다른 순서로 완성한 해법을 배제하지 않는다.

## 상태 관리

스테이지의 런타임 상태는 최소한 다음 요소로 구성된다.

- 모든 플레이 셀의 R/G/B 포함 상태
- 색상·범위별 남은 물감통 수량
- 입력 가능, 확산 연출 중, 완료 상태를 구분하는 진행 상태
- 되돌리기를 위한 행동 직전 스냅샷 이력

되돌리기는 직전 배치 전의 전체 셀 상태와 물감통 수량을 복원하며 여러 단계 연속 실행할 수 있어야 한다. 초기화는 이력을 포함한 런타임 상태를 스테이지 시작 데이터로 되돌린다. 정답 판정은 확산 연출과 결과 반영이 모두 끝난 뒤 각 셀의 표시 상태를 전체 비교하는 방식으로 수행한다.

## 스테이지 데이터

각 스테이지는 콘텐츠와 런타임 로직을 분리할 수 있도록 다음 항목을 데이터로 제공한다.

- 격자 크기
- 인접 셀 사이의 가로·세로 벽
- 각 셀의 목표 색상 상태
- 제공 물감통의 색상, 범위, 수량
- 잠금 및 완료 상태 등 스테이지 선택 화면용 정보

정답판과 플레이판은 같은 벽 데이터를 공유한다. 제작 도구에서는 정답 색상, 벽, 물감통 구성을 에디터에서 설정할 수 있어야 하며, 콘텐츠 제작 전에 핵심 규칙과 퍼즐 제작 도구를 먼저 검증한다.

## 개발 범위와 우선순위

프로그래밍의 핵심 범위는 격자 좌표와 셀 생성, 벽 기반 인접성 판정, BFS 확산, 색상 상태 및 W 처리, 물감통 선택·수량 관리, 순차 연출과 입력 잠금, 되돌리기·초기화, 정답 판정, 스테이지 데이터화, WebGL 대응이다.

아트의 핵심 범위는 정답판·플레이판과 벽, 범위가 표시된 R/G/B/W 물감통, 기본색·혼합색·흰색·빈 칸의 구분, 거리 단계별 확산 애니메이션, 조작 UI, 성공 연출, 스테이지 선택 화면이다.

개발 순서는 다음과 같다.

1. 격자, 배치, 단순 확산으로 핵심 규칙을 프로토타이핑한다.
2. 벽 경로 탐색, 색상 혼합, W 처리를 구현한다.
3. 물감통 수량, 되돌리기, 초기화 등 상태 관리를 완성한다.
4. 확산 애니메이션, 입력 잠금, 성공 판정을 연동한다.
5. 스테이지 제작 도구와 데이터 구조를 마련한다.
6. 튜토리얼과 본 스테이지를 제작하고 풀이를 검증한다.
7. 화면 비율, 클릭 판정, 로딩을 포함해 WebGL 빌드를 최적화한다.

전체 우선순위는 **핵심 규칙 검증 → 퍼즐 제작 도구 → 시각 연출 → 스테이지 콘텐츠 → WebGL 배포**다.
