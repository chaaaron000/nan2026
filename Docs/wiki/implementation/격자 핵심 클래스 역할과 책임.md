## Overview

현재 격자 구현은 논리 상태와 벽 이동 규칙인 `GridState`, 공통 좌표 변환인 `GridIndexUtility`, 방향 표현인 `GridDirection`, 화면 구성인 `GridView`, 개별 칸 표현과 입력인 `CellView`, RGB 상태 값인 `PaintState`로 나뉜다. 게임의 실제 색상과 벽 상태는 일반 C# 객체에 보관하고 Unity `MonoBehaviour`는 생성·배치·입력·표현을 맡는다.

## 전체 협력 구조

```text
물감통 행동 처리 계층(예정)
    ├─ GridState의 색상 상태 변경
    └─ GridView에 표현할 결과 전달
             └─ 해당 CellView의 색상 및 애니메이션 갱신

CellView 클릭
    └─ 논리 좌표 Vector2Int 전달
             └─ 물감통 행동 처리 계층에서 배치 시도
```

`GridState`와 View 계층 사이에는 논리 좌표인 `Vector2Int`를 공통 식별자로 사용한다. `CellView`는 자신의 Transform 위치로부터 좌표를 역산하지 않고, 생성 시 전달받은 `GridPosition`을 유지한다.

## GridState

`GridState`는 현재 플레이 중인 격자의 논리 상태를 보관하고 변경하는 일반 C# 클래스다. 씬 오브젝트나 Unity 생명주기가 필요하지 않으므로 `MonoBehaviour`를 상속하지 않는다.

현재 책임은 다음과 같다.

- 격자의 `Width`, `Height` 보관
- `PaintState[]` 배열에 모든 셀의 현재 색상 보관
- 2배 좌표계의 벽 위치를 `HashSet<Vector2Int>`로 보관
- `Vector2Int` 좌표가 격자 내부인지 판정
- `GridIndexUtility`를 통한 좌표와 배열 인덱스 변환
- 좌표의 현재 색상 조회
- OR 연산을 통한 RGB 물감 추가
- 커맨드 Undo를 위한 셀 상태 교체
- 한 칸 또는 전체 격자의 색상 제거
- 인접한 두 셀 또는 셀과 방향을 받아 이동 가능 여부 판정

`AddPaint`는 기존 값과 전달받은 색을 OR 연산하고 최종 `PaintState`를 반환한다. `SetPaint`는 커맨드가 실행 직전의 임의 RGB 조합을 정확히 되돌릴 수 있도록 전달받은 상태로 셀 값을 교체한다. `ClearPaint`는 W 물감의 삭제 규칙에서 사용할 수 있지만, W의 확산 범위나 물감통 소모 자체는 `GridState`가 결정하지 않는다.

`GridState`는 벽 존재와 한 칸 이동 가능 여부까지만 판정한다. 여러 셀을 탐색해 확산 범위를 결정하는 책임은 `PaintSpreadCalculator`에 있고, 물감통 소모와 색 적용 순서는 `PaintBucketUseCommand`에 있다. GameObject 생성, 색상 연출, 정답 판정도 맡지 않는다.

## GridIndexUtility

`GridIndexUtility`는 여러 격자 클래스가 공유하는 행 우선 좌표 규칙을 한곳에 모은 static 클래스다.

- `IsInside(position, width, height)`: 좌표 범위 판정
- `ToIndex(position, width, height)`: `y * width + x` 인덱스로 변환
- `ToPosition(index, width, height)`: 인덱스를 `(x, y)` 좌표로 복원
- 유효하지 않은 격자 크기, 좌표, 인덱스에 예외 발생

`GridState`의 셀 배열 접근, `GridView`의 CellView 조회, BFS의 방문 배열이 같은 변환 규칙을 사용한다. 벽의 2배 좌표계나 Transform 위치 변환은 별도 규칙이므로 이 유틸리티에 포함하지 않는다.

## GridDirection과 벽 이동

`GridDirection`은 `UP`, `RIGHT`, `DOWN`, `LEFT` 네 방향을 나타내고 `GridDirectionUtility.ToOffset`이 각 값을 한 칸짜리 좌표 변화로 바꾼다.

벽은 셀 중심 좌표를 2배로 확장한 좌표계에서 한 축만 홀수인 위치에 존재한다. 인접한 두 셀 `from`, `to`의 합은 이 좌표계에서 두 셀 사이의 벽 위치가 된다. `GridState.CanMove`는 다음을 확인한다.

1. 출발지와 목적지가 격자 안인지 확인한다.
2. 두 좌표의 맨해튼 거리가 1인지 확인한다.
3. `walls.Contains(from + to)`로 사이 벽이 없는지 확인한다.

방향 오버로드는 목적지를 계산한 뒤 두 좌표 오버로드에 판정을 위임한다. 벽을 `HashSet`으로 보관하므로 존재 여부 조회는 평균 O(1)이다.

## PaintState

`PaintState`는 셀 하나에 포함된 R, G, B 상태를 나타내는 값 타입이다. 각 기본색은 서로 다른 비트를 사용하며 혼합색은 기본색 조합의 별칭이다.

| 상태 | 비트 | 의미 |
|---|---:|---|
| `Empty` | `000` | 칠해지지 않은 빈 칸 |
| `Red` | `100` | R 포함 |
| `Green` | `010` | G 포함 |
| `Blue` | `001` | B 포함 |
| `Yellow` | `110` | R + G |
| `Cyan` | `011` | G + B |
| `Magenta` | `101` | R + B |
| `White` | `111` | R + G + B |

`[Flags]`는 비트 연산을 강제하거나 잘못된 값을 제한하지 않는다. 이 enum이 조합 용도라는 의도를 표시하고 조합 값의 문자열 표현과 일부 도구 표시를 돕는다. W는 셀에 남는 색상 상태가 아니라 기존 RGB를 제거하는 행동이므로 `PaintState`에 포함하지 않는다.

## GridView

`GridView`는 격자 전체를 화면에 만들고 관리하는 `MonoBehaviour`다. `GridState`의 크기를 읽어 각 좌표에 `CellView` 프리팹을 생성하고, 생성한 View를 같은 순서의 1차원 배열에 저장한다.

현재 책임은 다음과 같다.

- `GridState.Width`, `Height`에 맞춰 셀 생성
- `GridState.WallPositions`에 맞춰 테스트용 벽 오브젝트 생성
- 논리 좌표를 셀의 로컬 위치로 변환
- 생성한 `CellView` 참조 보관
- 좌표에 해당하는 CellView에 최종 PaintState 전달
- 격자를 다시 만들기 전에 기존 셀 제거

셀의 로컬 위치는 현재 `(x * cellSize, y * cellSize, 0)`으로 계산한다. 타일을 `GridView`의 자식으로 생성하므로 부모 Transform을 움직여 격자 전체 위치를 조절할 수 있다.

`SetCellPaint`는 `GridIndexUtility`로 논리 좌표에 해당하는 View를 찾아 최종 상태를 전달한다. 여러 셀에 걸친 확산 애니메이션의 시간 순서는 별도 상위 연출 계층에서 조율하고, `GridView`는 좌표와 View 사이의 연결을 담당한다.

## CellView

`CellView`는 격자의 개별 칸에 붙는 `MonoBehaviour`다. 한 칸의 화면 표현과 클릭 입력을 담당하며 실제 `PaintState`의 소유자가 아니다.

현재 책임은 다음과 같다.

- 생성 시 전달받은 `GridPosition` 보관
- 좌표를 포함한 GameObject 이름 설정
- 클릭 시 `Clicked` 이벤트로 논리 좌표 전달

현재 `SetPaint`가 `PaintState`를 임시 `SpriteRenderer.color`로 변환한다. 향후에는 애니메이션 없는 즉시 동기화와 한 칸의 칠하기·혼합·지우기 애니메이션을 분리할 수 있다.

`CellView`는 물감 혼합 결과나 확산 도달 여부를 직접 계산하지 않는다. 상위 논리 계층이 결정한 최종 `PaintState`를 받아 표현해야 한다. 또한 여러 셀의 연출 순서를 직접 관리하거나 다른 `CellView`를 탐색하지 않는다.

## 데이터와 화면의 일관성

게임의 권위 있는 상태는 `GridState`에 있다. `CellView`의 Sprite나 색상은 그 상태를 표현한 결과일 뿐이므로, 화면만 변경하고 `GridState`를 갱신하지 않는 흐름을 만들지 않는다.

권장되는 변경 순서는 다음과 같다.

1. 물감통 행동 계층이 영향을 받는 좌표를 계산한다.
2. 해당 좌표에 `GridState.AddPaint` 또는 `ClearPaint`를 호출한다.
3. 반환되거나 조회된 최종 `PaintState`를 `GridView`에 전달한다.
4. `GridView`가 좌표에 해당하는 `CellView`를 찾아 표시 또는 애니메이션을 요청한다.

초기 로드, 초기화, 되돌리기는 애니메이션 없이 화면을 즉시 동기화하는 API를 사용하고, 실제 물감통 사용에는 별도의 애니메이션 API를 사용하는 방향이 적절하다.

## 아직 결정하지 않은 영역

- 확산 애니메이션의 단계, 시간 및 입력 잠금 방식
- 목표 상태 저장과 정답 판정 객체

이 항목들은 현재 네 타입의 책임에 섞지 않고 각 기능을 구현할 때 별도로 구체화한다.
