# 물감 확산 퍼즐 기획서

## 1. 게임 개요

| 항목 | 내용 |
| --- | --- |
| 장르 | 색상 조합 기반 논리 퍼즐 |
| 플랫폼 | PC WebGL |
| 개발 환경 | Unity 2D |
| 핵심 조작 | 물감통 선택 후 격자 칸 클릭 |
| 플레이 목표 | 제한된 물감통을 배치해 정답과 동일한 색상 무늬 완성 |
| 주요 시스템 | 맨해튼 거리 확산, 벽, 색상 혼합, 지우기, 되돌리기 |
| 실패 조건 | 없음 |
| 정답 조건 | 현재 플레이 격자의 모든 칸이 정답 그림과 일치 |
| 핵심 경험 | 물감의 위치·범위·순서를 역으로 추론하는 과정 |

### 한 줄 소개

**정답 그림과 벽 구조를 관찰하고, 범위와 색상이 정해진 물감통을 순서대로 배치해 같은 색상 무늬를 완성하는 논리 퍼즐 게임.**

---

## 2. 게임 소개

게임 화면에는 동일한 벽 구조를 가진 두 개의 격자판이 배치된다.

- **정답판:** 플레이어가 완성해야 하는 색상 무늬 표시
- **플레이판:** 물감통을 직접 배치하고 결과를 확인하는 공간
- **물감통 목록:** 스테이지에서 사용할 수 있는 색상·범위·수량 표시
- **보조 기능:** 되돌리기와 초기화 제공

플레이어는 R, G, B, W 물감통 중 하나를 선택한 뒤 플레이판의 원하는 칸에 배치한다. 물감통은 배치된 칸에서부터 벽으로 막히지 않은 상하좌우 경로를 따라 지정된 범위만큼 퍼진다.

R, G, B 물감이 같은 칸에 겹치면 색상 조합이 발생한다.

| 물감 조합 | 결과 |
| --- | --- |
| R | 빨강 |
| G | 초록 |
| B | 파랑 |
| R + G | 노랑 |
| G + B | 청록 |
| B + R | 자홍 |
| R + G + B | 흰색 |

동일한 색이 여러 번 겹쳐도 색상은 변하지 않는다. 각 칸은 R, G, B의 포함 여부만을 기준으로 최종 색상을 표시한다.

W 물감은 칠해진 색상을 제거하는 특수 물감이다. W가 지나간 칸은 빈 상태가 되며, 이후 다른 물감을 다시 덧칠할 수 있다. 이에 따라 물감통의 배치 위치뿐 아니라 **사용 순서**도 풀이의 핵심이 된다.

### 기본 플레이 흐름

**정답 확인 → 벽 구조 분석 → 물감통 선택 → 칸 배치 → 확산 연출 → 색상 결과 확인 → 정답 비교**

현재 그림이 정답과 일치하면 물감 확산 연출이 끝난 뒤 성공 팝업이 출력된다. 제공된 물감통을 모두 사용하지 않았거나 개발자가 의도한 풀이와 다른 방법을 사용했더라도 최종 그림이 같으면 정답으로 인정한다.

## 3. 재미 요소와 차별점

### 역방향 추론

완성된 그림을 보고 물감통이 놓일 위치와 순서를 역으로 계산한다. 플레이어는 각 물감통이 만들어 낼 전체 결과를 예상하며 풀이한다.

### 벽을 활용한 경로 계산

물감은 이동 가능한 격자 경로를 따라 확산된다. 정면이 막혀 있어도 남은 범위가 있다면 열린 길을 통해 옆으로 돌아갈 수 있다.

### 위치와 순서를 함께 계산하는 퍼즐

R, G, B 물감들과 W 물감의 사용 순서도 결과에 영향을 준다.

**넓게 칠하기 → 일부 영역 지우기 → 필요한 색 다시 덧칠하기**

같은 위치에 물감통을 배치하더라도 사용 순서에 따라 다른 결과가 만들어진다.

### 다양한 정답 풀이 허용

개발자가 기준 풀이를 설계하더라도 최종 색상만 같으면 정답으로 인정한다. 더 적은 수의 물감통을 사용하거나 다른 배치 순서로 해결하는 창의적인 풀이가 가능하다.

### 즉각적인 시각 피드백

물감통을 놓으면 해당 칸에서부터 인접 칸으로 물감이 퍼지는 모습을 즉시 보여준다. 플레이어는 예상한 결과와 실제 결과를 비교하며 다음 배치를 수정한다.

### 부담 없는 시행착오

시간 제한과 게임 오버 없이 자유롭게 풀이할 수 있다. 잘못된 배치는 되돌리기로 한 단계씩 수정하고, 풀이 방향이 크게 어긋났을 때는 초기화를 사용한다.

## 4. 핵심 시스템 기획

### 4.1 격자 및 벽 시스템

- 정답판과 플레이판은 동일한 크기와 벽 구조 사용
- 벽은 칸과 칸 사이에 배치되는 고정 오브젝트
- 벽은 플레이 중 생성·제거·이동되지 않음
- 격자 외부는 이동할 수 없는 경계(벽)로 처리
- 정답판에는 목표 색상과 벽을 함께 표시

### 4.2 물감 확산 시스템

물감통의 범위가 N일 때, 배치 칸을 포함해 상하좌우로 최대 `N-1회` 이동하여 도달할 수 있는 모든 칸에 물감이 적용된다.

- 대각선 이동 불가
- 벽 통과 불가
- 벽을 우회하는 경로는 허용
- 동일한 칸에 여러 물감통 배치 가능
- 물감통은 배치 즉시 소모
- 사용한 물감통 오브젝트는 격자에 남지 않음
- 확산 연출 중에는 추가 입력 차단

### 4.3 색상 처리 시스템

각 칸은 R, G, B 세 가지 색상 포함 여부를 저장한다.

- 같은 색의 중복 적용은 결과에 영향 없음
- 서로 다른 색의 조합에 따라 최종 표시 색상 결정
- W 적용 시 해당 칸의 R, G, B 정보를 모두 제거
- W 적용 이후 새로운 물감 덧칠 가능
- 물감통 배치 순서에 따라 최종 결과 변경

### 4.4 물감통 제공 시스템

각 스테이지에는 사용할 수 있는 물감통이 고정 제공된다.

- 색상: R, G, B, W
- 범위: 물감통마다 개별 설정
- 수량: 스테이지별 물감통 개수가 정해짐
- 사용 시 해당 물감통 소모

### 4.5 되돌리기 및 초기화

**되돌리기**

- 직전 물감통 배치 이전 상태로 복원
- 전체 격자 색상 상태 복원
- 사용한 물감통 수량 반환
- 여러 번 사용해 배치 과정을 단계별로 복원

**초기화**

- 모든 플레이 격자 색상 제거
- 모든 물감통 수량 복원
- 스테이지 시작 상태로 복귀

### 4.6 정답 판정

물감 확산 연출이 완료될 때마다 플레이 격자와 정답 격자를 비교한다.

- 모든 칸의 색상이 같으면 성공
- 물감통 배치 위치와 순서, 남은 물감통 수량은 판정에 포함하지 않음
- 의도한 풀이와 달라도 최종 그림이 같으면 성공
- 성공 시 입력 차단 후 스테이지 완료 팝업 출력

## 5. 아트 / 프로그래밍별 개발 범위 및 계획

### 5.1 아트 개발 범위

| 작업 | 주요 내용 |
| --- | --- |
| 격자 디자인 | 정답판·플레이판 칸, 테두리, 빈 칸 표현 |
| 벽 디자인 | 가로·세로 벽 오브젝트, 연결부와 외곽 처리 |
| 물감통 디자인 | R, G, B, W 색상 구분 및 범위 숫자 표시 |
| 색상 타일 | 기본색, 혼합색, 흰색, 빈 칸 시각적 구분 |
| 확산 연출 | 시작점에서 인접 칸으로 퍼지는 물감 애니메이션 |
| UI 디자인 | 물감통 목록, 남은 수량, 되돌리기, 초기화 버튼 |
| 성공 연출 | 퍼즐 완료 강조 효과 및 성공 팝업 |
| 스테이지 UI | 스테이지 선택, 잠금 상태, 완료 상태 표시 |

#### 아트 제작 계획

1. **스타일 가이드 제작:** 격자 크기, 선 두께, 색상 팔레트 확정
2. **플레이 화면 제작:** 격자판, 벽, 물감통, 기능 버튼 구성
3. **색상 가독성 검증:** 흰색과 빈 칸, 혼합색 간 명확한 구분
4. **확산 애니메이션 제작:** 이동 거리 단계에 따른 순차 연출
5. **완료 UI 제작:** 성공 팝업과 다음 스테이지 이동 화면 구성

| 작업 | 주요 내용 |
| --- | --- |

| 순차 처리 | 물감 적용 순서와 입력 잠금 관리 |

#### 프로그래밍 개발 계획

1. **기본 프로토타입:** 격자 생성, 물감통 배치, 단순 범위 확산
2. **퍼즐 규칙 구현:** 벽 경로 탐색, 색상 혼합, W 처리
3. **상태 관리 구현:** 물감통 수량, 되돌리기, 초기화
4. **연출 연동:** 확산 애니메이션, 입력 잠금, 성공 판정
5. **스테이지 데이터화:** 에디터에서 정답·벽·물감통 구성 가능하도록 제작
6. **콘텐츠 제작:** 튜토리얼 및 본 스테이지 구성·검증
7. **WebGL 최적화:** 화면 비율, 클릭 판정, 로딩 및 브라우저 테스트
